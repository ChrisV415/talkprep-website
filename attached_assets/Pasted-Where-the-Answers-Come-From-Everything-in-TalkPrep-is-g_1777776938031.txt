Where the Answers Come From
Everything in TalkPrep is generated by Claude — the AI I am. When a user fills in their situation and hits "Generate my prep guide," the app sends their details directly to Anthropic's API and Claude writes the response in real time. The opening script, the three anticipated responses, the fallback plan, the grounding thought — all of it is Claude reasoning about their specific situation and generating language on the spot.
There is no database of pre-written scripts. There is no search happening. No external data is being pulled. It's pure AI generation every single time, which is why it can be specific to someone telling their mom they're not coming to Christmas versus someone resigning after 8 years.
The role-play mode works the same way — Claude receives the conversation history plus a system instruction saying "play this specific person" and responds in character. The debrief is another Claude call with the outcome details.

Is There a Free API for This?
Here's the honest answer — and the cost reality you need to know before building this as a business.
Claude API (Anthropic) — what the app currently uses
Not free. Costs roughly $0.003 per 1,000 tokens. A typical TalkPrep session (prep guide + role-play + debrief) uses about 2,000–4,000 tokens total. That's roughly $0.006–$0.012 per session — less than one cent per user interaction.
OpenAI API (GPT-4o)
Also not free. Similar pricing, similar quality for this use case.
"Free" options that actually exist:

The Real Cost Math for TalkPrep as a Business
This is the part that actually matters.
At Claude's current pricing, here's what a paying user costs you:

Single session ($4.99 price): API cost ~$0.01. Margin: ~$4.98 before Stripe fees.
Monthly subscriber ($12.99): If they use it 10 times a month, API cost ~$0.10. Margin: ~$12.89 before Stripe fees.
Annual subscriber ($79): If they use it 50 times a year, API cost ~$0.50. Margin: ~$78.50 before Stripe fees.

The AI cost is essentially negligible compared to what users pay. This is one of the structural advantages of building on top of large language models right now — the cost per inference is so low that your margins are almost entirely determined by how many paying users you have, not by compute costs.
The only cost scenario that hurts you is if you have a very generous free tier and thousands of people use it heavily without converting to paid. At 3 free sessions per user that's still only about $0.03 in API costs per free user — manageable until you're at serious scale.

What You Should Actually Do
Right now (prototype stage): The app already uses Claude. That's the right choice — best quality, most nuanced emotional language, handles personal conversation prep better than open-source alternatives.
When you launch: Keep Claude. The quality difference matters for this product specifically. Someone preparing for the hardest conversation of their year needs the best output, not the cheapest one. A slightly worse script because you switched to a free model is a product problem that shows up in reviews.
To reduce costs at scale: Anthropic has volume pricing. At meaningful scale ($10k+/month in API spend) you negotiate directly. You're not there yet and won't be for a while.
One thing to change in the app: Right now the API key is exposed in the frontend HTML. That works for a demo but not for production. When you deploy for real users you need a simple backend — even a one-file Node.js or Python server — that holds the API key securely and proxies requests. Without that, anyone can open your page's source code, steal your API key, and run up your bill. That's the one real technical thing to fix before charging real money.
